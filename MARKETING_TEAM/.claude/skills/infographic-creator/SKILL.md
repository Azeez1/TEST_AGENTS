## üöÄ Core Capabilities

### **1. Seven Infographic Types**

This skill supports all major infographic categories:

#### **a) Statistical Infographic**
üìä **Purpose**: Display numbers, percentages, metrics, KPIs
- **Best for**: Annual reports, survey results, metrics dashboards, ROI data
- **Elements**: Big numbers, percentage bars, charts (bar/pie/line), comparison tables
- **Templates**: `templates/statistical-template.html`
- **Example use**: "Create infographic showing Q4 metrics: 127% growth, 50K customers, 95% satisfaction"

#### **b) Timeline Infographic**
üìÖ **Purpose**: Show chronological events, milestones, history
- **Best for**: Company history, project roadmaps, product launches, historical events
- **Elements**: Timeline axis, milestone markers, date labels, connecting lines
- **Templates**: `templates/timeline-template.html`
- **Example use**: "Create timeline showing our product roadmap Q1-Q4 2025"

#### **c) Comparison Infographic**
‚öñÔ∏è **Purpose**: Side-by-side comparisons, before/after, versus
- **Best for**: Product comparisons, A/B test results, competitive analysis, feature tables
- **Elements**: Split layouts, comparison tables, pros/cons lists, side-by-side visuals
- **Templates**: `templates/comparison-template.html`
- **Example use**: "Create comparison infographic: Our product vs Competitor A vs Competitor B"

#### **d) Process Infographic**
üîÑ **Purpose**: Step-by-step flows, how-to guides, workflows
- **Best for**: Tutorials, customer journeys, onboarding flows, manufacturing processes
- **Elements**: Numbered steps, arrows, icons, flow indicators
- **Templates**: `templates/process-template.html`
- **Example use**: "Create process infographic showing our 5-step sales methodology"

#### **e) Hierarchical Infographic**
üî∫ **Purpose**: Pyramids, org charts, nested data, priority frameworks
- **Best for**: Organization structure, Maslow's hierarchy, priority matrices, taxonomies
- **Elements**: Pyramid shapes, tree diagrams, nested boxes, org chart connectors
- **Templates**: `templates/hierarchical-template.html`
- **Example use**: "Create pyramid showing our product feature priorities"

#### **f) Geographic Infographic**
üó∫Ô∏è **Purpose**: Maps, locations, regional data, spatial information
- **Best for**: Market expansion, demographic data, store locations, global reach
- **Elements**: Maps (world/continent/country), markers, heat maps, choropleth overlays
- **Templates**: `templates/geographic-template.html`
- **Example use**: "Create map showing our customer distribution across 50 states"

#### **g) List Infographic**
üìã **Purpose**: Tips, features, facts, checklists, rankings
- **Best for**: Top 10 lists, feature highlights, quick tips, key takeaways
- **Elements**: Numbered/bulleted lists, icons, visual separators, highlight boxes
- **Templates**: `templates/list-template.html`
- **Example use**: "Create infographic: 10 Ways to Boost Your Marketing ROI"

## ü§ñ AI Design Intelligence

### **Automatic Design Optimization**

The skill applies professional design principles automatically:

#### **1. Color Intelligence**
```
Sentiment-based color selection:
- Positive metrics (growth, revenue) ‚Üí Greens, golds
- Negative metrics (churn, costs) ‚Üí Reds, oranges
- Neutral data ‚Üí Blues, grays
- Comparison sets ‚Üí High-contrast pairs

Accessibility checking:
- Minimum 4.5:1 contrast ratio (WCAG AA)
- Color blindness simulation (deuteranopia, protanopia)
- Auto-adjust if contrast fails
```

#### **2. Layout Intelligence**
```
Data density optimization:
- Low data (1-5 points) ‚Üí Large, centered, spacious
- Medium data (6-15 points) ‚Üí Grid layout, balanced
- High data (16+ points) ‚Üí Compact, hierarchical, grouped

Composition rules:
- Rule of thirds for focal points
- Visual balance (left/right weight)
- Whitespace calculation (minimum 20% negative space)
- Golden ratio for proportions
```

#### **3. Typography Hierarchy**
```
Automatic font scaling:
- Headline: 48-72px (main message)
- Subheadline: 32-40px (supporting info)
- Body: 18-24px (details)
- Labels: 12-16px (annotations)

Responsive scaling:
- Auto-adjust for canvas size
- Prevent text overflow
- Maintain readability at all export sizes
```

#### **4. Visual Hierarchy**
```
Attention flow optimization:
- Primary focal point (largest, highest contrast)
- Secondary elements (medium emphasis)
- Tertiary details (subtle, supporting)

Eye path guidance:
- Z-pattern for text-heavy infographics
- F-pattern for list infographics
- Circular flow for process infographics
```

## üìñ Best Practices & Guidelines

### **Design Principles**

#### **1. Data-Ink Ratio**
```
Maximize data, minimize decoration:
‚úÖ Every element serves a purpose
‚úÖ Remove chartjunk and clutter
‚úÖ Let data speak for itself
‚ùå Avoid unnecessary 3D effects
‚ùå Don't over-decorate
```

#### **2. Visual Hierarchy**
```
Guide attention intentionally:
1. Primary message (largest, highest contrast)
2. Supporting data (medium emphasis)
3. Context/details (subtle, smaller)
```

#### **3. Color Coding**
```
Use color meaningfully:
‚úÖ Consistent color = consistent meaning
‚úÖ High contrast for important data
‚úÖ Analogous colors for related items
‚úÖ Complementary colors for comparisons
‚ùå Don't use random colors
‚ùå Avoid more than 5-6 colors
```

#### **4. Typography**
```
Hierarchy and readability:
- One headline font
- One body font
- Maximum 3 font sizes per infographic
- Minimum 18px for body text
- High contrast (text vs background)
```

#### **5. Whitespace**
```
Breathing room matters:
- Minimum 20% negative space
- Margins around all elements
- Generous padding
- Visual separation between sections
```

### **When to Use Each Type**

| Content | Recommended Type | Why |
|---------|------------------|-----|
| Metrics, KPIs, numbers | Statistical | Shows data clearly with charts |
| Historical events | Timeline | Chronological structure |
| Two+ options | Comparison | Side-by-side clarity |
| How-to, workflows | Process | Sequential steps |
| Organization structure | Hierarchical | Shows relationships |
| Regional data | Geographic | Spatial understanding |
| Tips, features | List | Scannable, digestible |

### **Animation Best Practices**

```
‚úÖ Keep animations under 10 seconds
‚úÖ Use subtle, smooth easing (avoid bouncy)
‚úÖ Animate elements sequentially (not all at once)
‚úÖ Provide still frame as backup (for accessibility)
‚úÖ Test on mobile (performance)

‚ùå Don't over-animate (distracting)
‚ùå Don't animate critical text (hard to read)
‚ùå Don't use jarring transitions
```

### **Accessibility Guidelines**

```
‚úÖ Minimum 4.5:1 contrast ratio (text vs background)
‚úÖ Alt text for all images
‚úÖ Color is not the only differentiator (use patterns too)
‚úÖ Provide text alternative for data
‚úÖ Support keyboard navigation (interactive infographics)
‚úÖ Test with color blindness simulators
```

## üîß Troubleshooting

### **Common Issues**

**Problem: Text is too small / hard to read**
```
Solution:
--min-font-size 18
--scale 1.5  # Increase overall scale
```

**Problem: Colors look washed out**
```
Solution:
--contrast high
--saturation 120  # Increase color intensity
```

**Problem: Animation is choppy**
```
Solution:
--animation-fps 60  # Higher frame rate
--reduce-complexity  # Simplify animation
--duration longer  # Slower = smoother
```

**Problem: File size too large for social media**
```
Solution:
--compress aggressive
--optimize-for linkedin|twitter|instagram
--reduce-quality 85  # Still looks great
```

**Problem: Chart is cluttered / too busy**
```
Solution:
--max-data-points 10  # Limit displayed data
--group-small-values  # Combine small slices
--simplify  # AI removes non-essential elements
```
